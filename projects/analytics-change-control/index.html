01 · Problem

“Can you add this?” is not a specification.

Analytics teams lose time when requests arrive without an owner, business decision, KPI definition, source, priority, acceptance criteria, or approval path.

The workflow converts an idea into a reviewable delivery contract while keeping urgent support work distinct from planned enhancement work.

Intake

Ambiguity

A requested chart may hide a data-source gap, metric conflict, or process change.

Delivery

Invisible tradeoffs

Priority, dependency, and SLA decisions must be explicit to stakeholders.

Release

Unproven completion

“Built” is not “done” until data QA, acceptance, documentation, and release notes are complete.

02 · Workflow

One path from request to release.

Each stage has an accountable output. Work cannot silently jump from idea to dashboard.

03 · Example request

Definition before development.

CHANGE-024 is synthetic but complete enough to demonstrate the working standard.

CHANGE-024 · Add assisted conversion view

P3 · Ready for QA
Business decision

Where should supporting channels receive credit?

Marketing needs a secondary planning view. The existing primary attribution KPI remains unchanged.

Scope

New view, no KPI replacement

Use governed channel mapping, 30-day lookback, EUR revenue, and deduplicated conversion paths.

Acceptance criteria

Four release conditions

  1. Matches reference query within 1%
  2. Primary KPI unchanged
  3. Unknown channel under 1%
  4. Definition visible in dashboard
QA plan

Reconcile, segment, regress

Compare totals, inspect channel distributions, validate edge cases, and run regression checks on existing pages.

04 · Governance

SLA is a promise with conditions.

Priority does not replace discovery. The clock begins when the request has a decision owner, minimum source context, and an agreed target outcome.

P3

10 target days

Material business impact or a broken reporting workflow with a defined owner.

P4

20 target days

A planned improvement with measurable value and manageable dependencies.

P5

30 target days

A low-risk optimization or backlog candidate that does not block a decision.

Gate

Five release checks

Scope approved, criteria met, data QA passed, stakeholder sign-off captured, release notes written.

Policy notice

Priorities and targets are an original portfolio framework for demonstration. They do not represent any employer's internal SLA policy.
