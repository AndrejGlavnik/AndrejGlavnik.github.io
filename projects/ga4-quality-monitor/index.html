01 · Problem

A green dashboard can still be wrong.

Visual QA only proves that a chart renders. It does not prove event coverage, uniqueness, campaign mapping, schema stability, freshness, or agreement with raw data.

This case study makes analytics quality observable as a product: each rule has an owner, threshold, severity, evidence, and next action.

Completeness

Did everything arrive?

Required events, dimensions, dates, and consent context must be present.

Validity

Does it follow the contract?

Types, taxonomies, identifiers, currencies, and values must meet agreed rules.

Reconciliation

Does the output agree?

Raw events and dashboard aggregates need a defined variance threshold.

02 · Data contract

Quality starts with a definition.

The contract establishes grain, required fields, freshness, ownership, commerce rules, and reconciliation tolerance before any alert is evaluated.

Grain & freshness

One row per event

Daily event coverage must complete within four hours. Missing dates are critical because every downstream comparison inherits the gap.

MAX(event_timestamp) ≥ now() - 4 hours
COUNT(DISTINCT event_date) = expected_dates
Commerce integrity

Transaction IDs are unique

Purchase revenue is only reliable when transaction identifiers, currency, and item-level joins are valid.

SELECT transaction_id
FROM purchases
GROUP BY 1 HAVING COUNT(*) > 1
Campaign governance

Source and medium are mapped

Unexpected values are separated from known channels and routed to Marketing Operations for correction.

Reconciliation

Variance stays under 1.5%

Dashboard aggregates are compared with governed raw queries at the same grain, date range, timezone, and currency.

Data notice

All values in this case study are synthetic and created for portfolio demonstration. No employer, customer, or production data is included.

04 · Delivery

The alert is only the start.

A useful quality system links the failed check to impact, evidence, ownership, and a repair path. Severity is based on business risk, not on how dramatic the chart looks.

P0

Protect decisions first

Revenue duplication, missing dates, and broken consent context block trusted reporting.

P1

Preserve evidence

Each run records query logic, threshold, affected rows, timestamps, and example values.

P2

Route by ownership

Tracking, data platform, marketing operations, and BI issues go to the team able to fix the source.

P3

Close with reconciliation

A fix is complete only after the rule passes and downstream output agrees within tolerance.
